# Starter prompts

Each prompt is ready to copy and paste. Replace bracketed text; do not paste private data into shared or tracked files.

## 1. First Clutch Brainstorm

```text
Act as Hatching Ground. Generate 5 small personal agent ideas from this input: [life area, recurring friction, project, repo list, routine, or “I want to build something small but I’m not sure what”]. Favor boring-but-useful, file-first ideas for one user. For each, provide the complete idea-card fields, state assumptions, identify privacy level, and define a first-run test. Avoid generic assistants, platforms, and premature integrations.
```

## 2. Shrink This Idea

```text
Shrink this idea to its smallest useful version: [idea]. Identify excess users, workflows, data sources, integrations, autonomy, and infrastructure. Return one narrow MVP, what it defers, a first-run test, and whether the original should be split. Do not design the full architecture yet.
```

## 3. Pick the Next Hatchling

```text
Compare these synthetic or safely summarized idea cards: [cards]. Score each from 1–5 for personal usefulness, MVP clarity, build simplicity, privacy safety, reuse potential, maintenance lightness, and fun/momentum. Show totals and important tradeoffs, then recommend exactly one next action using an approved recommendation label. Do not let the total override a serious privacy or clarity problem.
```

## 4. Turn This Into an Idea Card

```text
Convert this rough idea into the exact Hatching Ground idea-card template: [idea]. Ask at most one question, and only if the answer materially changes safety or the recommendation. Otherwise proceed with clearly labeled assumptions. Keep the smallest useful version testable in one first-run scenario.
```

## 5. Score This Clutch

```text
Score this clutch using the Hatching Ground incubation scorecard: [cards]. Give every criterion a 1–5 score with one-sentence evidence, identify uncertainty, apply privacy and hatch gates, and assign one approved recommendation label to each idea. Use synthetic or abstracted details only.
```

## 6. Prepare an Architecture Brief

```text
Check this selected idea against the Hatching Ground hatch gate, then fill the architecture-brief template: [idea card]. If a gate fails, stop before architecture and recommend the smallest clarification or split. Keep full architecture and implementation as separate later steps.
```

## 7. Check Public/Private Boundary

```text
Classify this idea and its artifacts as private-only, public-safe, or potentially publishable later: [safe summary]. Identify prohibited public content, propose synthetic replacements, and apply the public-release checklist. Do not reproduce sensitive source details in the answer.
```
