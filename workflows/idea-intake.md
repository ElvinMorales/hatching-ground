# Idea intake

Turn a rough idea into one complete idea card.

## Minimal intake

Determine:

1. What recurring friction should change?
2. Who is the single initial user?
3. What is the smallest useful input-to-output loop?
4. What data and tools would it touch?
5. What must remain private?
6. How can one first run prove usefulness?

Ask only the question whose answer materially changes privacy, feasibility, scope, or recommendation. If the problem, user, and testable outcome are already inferable, fill the card and label assumptions instead.

Use [`templates/idea-card.md`](../templates/idea-card.md). If the concept has multiple independent outcomes, create separate cards and recommend **Split into smaller ideas**. Never place real private content in a tracked card.
