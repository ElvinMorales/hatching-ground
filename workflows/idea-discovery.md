# Idea discovery

## Inputs

Start from one of: a life area, recurring friction, current project, repository list, routine, or “I want to build something small but I’m not sure what.” Summarize repository names or private contexts rather than copying sensitive content.

## Method

1. Find repeated manual steps, decisions that can be drafted for review, or information that needs a consistent shape.
2. Generate candidates with one user, one problem, and one useful output.
3. Prefer existing local files and read-only inputs over new integrations.
4. Shrink candidates until a first run can demonstrate value.
5. Classify privacy exposure and identify likely artifacts.

Reject vague ideas such as “an assistant for my life.” Replace them with a named input, transformation, output, and trigger. Do not propose platforms, broad autonomy, or infrastructure as the idea itself.

## Output

Return each candidate using the exact fields in [`templates/idea-card.md`](../templates/idea-card.md). A clutch should normally contain 3–7 cards. State assumptions and add a one-sentence first-run test in notes when using structured data.
